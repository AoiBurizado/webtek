User testing

Our Google Form includes the four main sections:

  1. General information:
    This section asked for the platform type, browsertype, age, gender and profession.
  2. Usefulness:
    This section questioned the user about how useful she or he felt the website was, if the website met the testers needs, and if it did everything the tester was expecting it to do.
  3. Ease of use:
    This section asked about the use, if it was easy for the user to navigate thorugh the website, if using ti was effortless and if the tester didn't notice any inconsistencies while using it.
  4. Ease of learning:
    The ease of Learning is questioning about how long it took for the user to learn how to use it.
  5. Satisfaction:
    The satisfication section is asking about how satisfied the testers are with the design and the way you navigate.

Feedback Summary

After reading though the responses from our usertest we found out that our user were very happy with the usefullness of the site. They did however have some problems with the search. They thought that it was confusing that the search was on sites that the search did not work. The search function used to be placed at the navigation bar as shown in P2-document design. On the pages it could not be used, we make the background field grey instead of the usual white. And some of them thought that it was confusing that the pictures did not link.
There where no complaints about the "ease of use". We got a good score from the test. Same for "ease of learning", and "satisfaction". Only bad score we got was from our users that uses a mac computer, they thought that the site was not pleasant to use. Most of our users used a windows computer with a chrome browser.

Our response to the feedback

We looked at the feedback we got from the users and changed realized that some of the users where testing an old version of the site. We were not completely done with it, but decided it would be nice to get some user feedback in case there was any big misunderstandings in the layout. As mentioned in the "Feedback Summary" the users had some issues with the "Search"-function and the "imagelink". The Imagelink was really easy to fix. The "Search"-function however was a bit different. The "search" function are actually not a "search"-function, but more like a "filter"-function. It filters the reviews on the site it is on. To solve this userconfusion, we moved the location of the "search/filter"-bar to the right of the page title. Which means this "search/filter" will only be on the index.html, dating_sim.html, rpg.html and visual_novel.html.
After testing the site with both new and old testsubjects again and showing them the improved site they were happy that we had thought of their complaints even if we had planned on changing them at the beginning. It seemed most of the testers, both live testers and "online" testers seemed to be happy with the design. They were also very excited about the konamicode which a lot found very creative.
